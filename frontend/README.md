# লাইফব্রিজ (LifeBridge) — ফ্রন্টএন্ড

জরুরি মুহূর্তে কোন হাসপাতালে বেড, আইসিইউ বা ভেন্টিলেটর খালি আছে তা লাইভ দেখানোর জন্য Next.js (App Router) ফ্রন্টএন্ড। পুরো UI বাংলায়।

## চালানোর নিয়ম

```bash
npm install
npm run dev
```

তারপর http://localhost:3000 এ যান।

## গঠন (Structure)

- `app/page.tsx` — হোম পেজ: শহরের লাইভ বেড/আইসিইউ কাউন্ট, সার্চ, কাছের হাসপাতাল
- `app/hospitals/page.tsx` — সব হাসপাতালের তালিকা + ফিল্টার
- `app/hospitals/[id]/page.tsx` — একটি হাসপাতালের বিস্তারিত পেজ (বেড, আইসিইউ, ব্লাড ব্যাংক, বিভাগ)
- `app/emergency/page.tsx` — জরুরি পেজ: নিকটতম হাসপাতাল, কল, অ্যাম্বুলেন্স
- `app/dashboard/patient/page.tsx` — রোগীর ড্যাশবোর্ড (পছন্দের হাসপাতাল, নোটিফিকেশন)
- `app/dashboard/hospital/page.tsx` — হাসপাতাল স্টাফ প্যানেল (ম্যানুয়াল বেড আপডেট)
- `app/dashboard/admin/page.tsx` — অ্যাডমিন ড্যাশবোর্ড (সিটি-ওয়াইড পরিসংখ্যান)
- `components/` — Navbar, Footer, HospitalCard, BedStatCard, LiveIndicator (লাইভ পালস সিগনেচার), EmergencyButton
- `lib/mock-data.ts` — ডামি ডেটা (৫টি হাসপাতাল)। ব্যাকএন্ড রেডি হলে এটাকে API কল দিয়ে replace করুন — টাইপ (`lib/types.ts`) একই রাখলে বাকি UI অপরিবর্তিত থাকবে।

## ডিজাইন টোকেন

- রঙ: ক্লিনিক্যাল নেভি ব্লু (#1E3A5F বেস) বিশ্বাসের জন্য, ভাইটাল গ্রিন (#16A34A) খালি বেডের জন্য, ক্রিটিক্যাল রেড (#DC2626) জরুরি/শূন্য অবস্থার জন্য, কশন অ্যাম্বার (#D97706) সীমিত availability-র জন্য।
- ফন্ট: হেডিং-এ Baloo Da 2 (বাংলা সাপোর্টসহ, শক্তিশালী-বন্ধুত্বপূর্ণ), বডিতে Hind Siliguri (উচ্চ পঠনযোগ্যতা), সংখ্যায় Roboto Mono (মনিটর/লাইভ ডেটার অনুভূতি দিতে)।
- সিগনেচার এলিমেন্ট: "লাইভ পালস" — ECG-স্টাইল অ্যানিমেটেড লাইন যা প্রতিটি লাইভ-স্ট্যাটাস মুহূর্তে ব্যবহৃত হয়, হাসপাতালের ভাইটাল সাইন মনিটরের রেফারেন্স।

## পরবর্তী ধাপ (Backend integration)

- `lib/mock-data.ts`-এর জায়গায় একটি `lib/api.ts` বানিয়ে real API/Firestore/Postgres থেকে ডেটা আনুন।
- লাইভ আপডেটের জন্য WebSocket বা polling (SWR/React Query) যোগ করুন যাতে "কত মিনিট আগে হালনাগাদ" প্রকৃত রিয়েল-টাইম হয়।
- Google Maps এম্বেড করে `app/hospitals/page.tsx`-এ ম্যাপ ভিউ যোগ করা যাবে।
- Authentication (রোগী/হাসপাতাল/অ্যাডমিন রোল) যোগ করতে NextAuth ব্যবহার করা যেতে পারে।
